根据对 docs 目录中 tool calling 相关文件的搜索与读取，关键结论如下：

1. **核心职责划分**：
   - **模型层**：负责决定是否调用工具（决策层）。
   - **工具层**：负责校验和执行工具调用（执行层）。
   - **运行时层**：负责维护消息闭环，确保工具调用与响应在对话流中正确流转。

2. **工具编排原则**：
   - 模型需先决定调用哪个工具，再由运行时执行工具并将 `ToolMessage` 返回给模型。
   - 此机制支持工具链的灵活编排与动态调用。

3. **验证文件**：
   - `docs/tool_calling.md` 明确定义了上述三层架构。
   - `docs/search_skill_demo.md` 提供了工具编排的实际演示场景。

综上，Tool Calling 系统采用“模型决策 - 工具执行 - 运行时闭环”的分层架构，确保工具调用的准确性与对话连贯性。
