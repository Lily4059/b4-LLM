# Tool Calling 架构核心机制总结

根据 `docs/tool_calling.md` 文档，Tool Calling 的架构遵循以下核心原则：

1. **职责分离**：
   - **模型层**：负责**决策**是否调用工具，以及选择哪个工具。
   - **工具层**：负责**校验参数**并**执行**具体操作。
   - **运行时层**：负责维护**消息闭环**，确保工具调用与响应的完整性。

2. **工具编排流程**：
   - 当用户询问特定任务（如“检索演示 工具编排”）时，系统应优先调用搜索工具验证文档。
   - 工具编排强调**模型先决定调用哪个工具**，随后由**运行时执行工具**并将 `ToolMessage` 返回给模型，形成完整的交互闭环。

这一机制确保了工具使用的规范性与消息流的清晰性，是构建可靠智能体系统的基础。
