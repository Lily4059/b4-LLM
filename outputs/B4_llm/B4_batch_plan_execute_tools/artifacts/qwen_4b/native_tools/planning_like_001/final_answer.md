基于对 `docs/tool_calling.md` 文件的分析，以下是关于 Tool Calling 机制的关键总结：

**核心职责划分：**
1.  **模型层 (Model)**：负责**决策**。即判断当前上下文是否需要调用工具，以及应该调用哪个工具。
2.  **工具层 (Tool Layer)**：负责**校验与执行**。接收模型的调用请求，验证参数合法性，并实际执行操作。
3.  **运行时层 (Runtime)**：负责**维护闭环**。管理消息流，将工具执行的结果（ToolMessage）返回给模型，确保对话上下文完整。

**工具编排流程：**
整个流程强调“先决策，后执行”：
1.  模型分析输入，决定调用特定工具。
2.  运行时执行该工具。
3.  工具返回结果（ToolMessage）给模型。
4.  模型基于结果继续生成回复。

这一机制确保了 AI 能够灵活地结合外部能力（如搜索、计算等）来完成任务，同时保持对话逻辑的连贯性。
