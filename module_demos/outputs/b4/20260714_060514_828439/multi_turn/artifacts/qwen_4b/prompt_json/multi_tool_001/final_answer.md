### 文档关注点差异对比

**共同关注点**：
1. 工具在 Agent 系统中的核心作用（执行能力）
2. 模型与工具之间的协作关系
3. 系统架构的组成要素

**关键差异点**：
| 维度 | agent_intro.txt | tool_calling.md |
|------|-----------------|-----------------|
| **系统架构** | 强调模型、工具、记忆、执行循环四要素 | 聚焦模型、工具层、运行三层架构 |
| **工具定位** | 工具是 Agent 的扩展能力（读取文件、计算） | 工具是独立校验执行层，与模型决策分离 |
| **记忆机制** | 明确 Memory 提供全局知识和历史上下文 | 未提及记忆，强调运行时消息闭环 |
| **职责划分** | 工具执行具体任务 | 模型决策、工具执行、运行时维护闭环 |

**结论与建议**：
- `agent_intro.txt` 更偏向**系统概念介绍**，适合初学者理解 Agent 整体架构。
- `tool_calling.md` 更侧重**工程实现细节**，强调工具与模型的职责分离和运行时约束。
- 建议在实际开发中结合两者：用前者理解系统全景，用后者指导工具调用逻辑设计。
